# Deploying CloakRFQ to Canton DevNet

Run the web app against the **real Canton DevNet** (the shared Five North / Seaport
validator) instead of the local sandbox. Same contracts, same UI — the only
difference is auth + a remote participant.

> TL;DR: fill in `web/.env.local`, `dpm build`, run `bootstrap-devnet.py`, start the
> web app. The app talks to DevNet through a server-side proxy that injects the token.

---

## Prerequisites

- The local toolchain (Daml `dpm`, Node) — see the main README / RUNBOOK.
- **Seaport DevNet credentials** (from the Seaport "Validator Development Access" doc):
  - Ledger API: `https://ledger-api.validator.devnet.sandbox.fivenorth.io`
  - OIDC token URL: `https://auth.sandbox.fivenorth.io/application/o/token/`
  - `client_id`: `validator-devnet-m2m`
  - `client_secret`: provided with the Seaport access details (shared secret — kept out of the repo)

---

## 1. Configure credentials (server-side only)

```bash
cp web/.env.local.example web/.env.local
# edit web/.env.local → set CLOAKRFQ_OIDC_CLIENT_SECRET=<the shared secret>
```

`web/.env.local` is **gitignored** — the secret never gets committed and never reaches
the browser (the proxy uses it server-side).

## 2. Build the DAR

```bash
(cd ledger && dpm build)
```

## 3. Bootstrap DevNet — upload DAR, allocate parties, write config

```bash
set -a; . web/.env.local; set +a        # load the creds into your shell
python3 scripts/bootstrap-devnet.py
```

This exchanges the OIDC creds for a token, uploads the DAR, allocates the 9 parties
(`cloakrfqSeller`, `cloakrfqFunderA`, …), grants the ledger user `CanActAs` for each,
and writes `web/public/ledger-config.json`.

- For a **fresh, isolated deal** (DevNet is persistent — you can't wipe it), add a run
  tag: `python3 scripts/bootstrap-devnet.py mytag` → parties suffixed `-mytag`.

## 4. Run the app

```bash
npm --prefix web install                 # first time only
npm --prefix web run dev                 # open the printed URL
```

It's now live on Canton DevNet. Build a deal in the UI; each action is a real on-ledger
transaction.

---

## How it works

- The browser calls `/v2/*` **same-origin**. The Next.js route handler
  (`web/app/v2/[...path]/route.ts`) proxies each call to the DevNet validator, injecting
  an OIDC **client-credentials Bearer token** (cached, auto-refreshed ~every 8h). The
  **client secret stays server-side.**
- `web/public/ledger-config.json` maps the 9 roles → DevNet party ids + `userId` +
  `packageRef`. It's generated by `bootstrap-devnet.py` (gitignored).

## Self-service deploy (Level 1) — let others test it without you

For a shared / Vercel link where **each visitor gets their own isolated deal** (no one
running `bootstrap-devnet.py`), enable per-session provisioning:

1. **One-time:** upload the DAR once (`bootstrap-devnet.py`, or any prior run has already
   done it — the package is shared on the validator).
2. Set these env vars (on Vercel: Project → Settings → Environment Variables), in addition
   to the `CLOAKRFQ_OIDC_*` + `CLOAKRFQ_LEDGER_TARGET` above:
   ```
   CLOAKRFQ_SESSION_PROVISIONING=1
   CLOAKRFQ_LEDGER_USER_ID=6
   CLOAKRFQ_PARTY_NAMESPACE=<the participant namespace, e.g. 1220a14ca128…>
   ```
3. Deploy. On first load, `web/app/api/session/route.ts` discovers a connected
   synchronizer, allocates a fresh versioned party set on that synchronizer, and waits
   until every party has submission permission before returning the UI configuration.
   This can take up to about 20 seconds. Reloads reuse the same parties; the topbar
   **"↻ New deal"** action provisions another isolated set after confirmation.

Notes: it's a public write endpoint (allocates parties on the shared validator) — it has a
crude per-instance guard, but add real rate-limiting before wide sharing. When the flag is
**off**, the app uses the single static `ledger-config.json` (the flow above) — unchanged.

## Toggle local ↔ DevNet

| Mode | How | Notes |
|---|---|---|
| **DevNet (shared deal)** | `web/.env.local` present + `bootstrap-devnet.py` | real network, auth'd, one shared deal |
| **DevNet (self-service)** | add `CLOAKRFQ_SESSION_PROVISIONING=1` (+ USER_ID, PARTY_NAMESPACE) | per-visitor isolated deals |
| **Local**  | remove/rename `web/.env.local` + `./scripts/reset-local-demo.sh` | fresh local ledger and parties; starts the UI |

## Reset the demo

- **DevNet** (persistent): fresh parties — `python3 scripts/bootstrap-devnet.py <newtag>`, then hard-reload.
- **Local**: `./scripts/reset-local-demo.sh` (true wipe), then hard-reload.

## Seeing the transactions

- **In-app** — `/activity` (persistent tx explorer: `updateId`, offset, created/archived,
  "Verify on ledger") and `/ledger` (per-party active contracts). These are the *only*
  place your confidential contracts are visible.
- **Network explorer** — [5N Lighthouse](https://lighthouse.devnet.cantonloop.com)
  shows validators / Canton Coin / traffic, but **not** your app's private contracts (by design).

## Gotchas

- **Token** expires ~8h; the proxy refreshes it automatically — no action needed.
- **Traffic** (the per-tx cost, paid in Canton Coin) is covered by the shared validator's
  balance — you do **not** need to request tokens from the foundation for a normal demo.
- **"party already exists"** on re-run is fine — the script resolves the deterministic id.
- **Secret hygiene** — keep the client secret only in `web/.env.local`; rotate it if exposed.
